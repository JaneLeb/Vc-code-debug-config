WSL (Windows Subsystem for Linux) — это, по сути, «Linux внутри Windows». Она позволяет запускать полноценную командную строку Ubuntu (или другого дистрибутива) прямо в обычном окне Windows, без всяких виртуальных машин.
Для студента «Школы 21» это самый важный инструмент после самого VS Code, и вот почему:
Родная среда: Робот Вертер и все проверки в школе работают на Linux. WSL позволяет вам писать и тестировать код в точно такой же среде, не выходя из Windows.
Настоящий Bash: Все ваши скрипты .sh и тесты будут работать именно так, как они будут работать при сдаче проекта.
Valgrind и др.: Инструменты для поиска утечек памяти (Valgrind), которые обязательны в школе, не работают на Windows. В WSL они запускаются без проблем.
Интеграция с VS Code: Вы нажимаете одну кнопку, и ваш VS Code «ныряет» внутрь WSL. 
Вы редактируете файлы в красивом интерфейсе Windows, а компиляция и запуск происходят в «настоящем» Linux.
Если вы установите WSL, вам не нужно будет возиться с компилятором cl.exe от Microsoft. 
Вы просто напишете в терминале sudo apt install gcc make, и у вас будет классический набор инструментов, как в школе.
Если у вас уже установлена WSL (Ubuntu), то вы обладаете самым мощным инструментом для обучения в «Школе 21».
Раз она есть, ваш рабочий процесс в VS Code может стать еще круче. Вот как это проверить и использовать прямо сейчас:
1. Как «зайти» в Linux из VS Code
Посмотрите в левый нижний угол окна VS Code. Видите там синий квадратик (или значок ><)?
Нажмите на него.
Выберите в появившемся меню: Connect to WSL (или Connect to WSL using Distro... и выберите Ubuntu).
VS Code перезагрузится. Теперь в синем квадратике внизу будет написано WSL: Ubuntu.
2. Что это меняет?
Теперь ваш VS Code «живет» внутри Linux:
Терминал: Если вы нажмете Ctrl + ~, откроется настоящий Bash (Ubuntu), а не Windows PowerShell.
Компилятор: Теперь вы можете использовать gcc, как в школе. Просто введите в терминале gcc --version, чтобы проверить.
Пути: Ваши файлы теперь будут лежать в файловой системе Linux (что работает гораздо быстрее для компиляции).
3. Нужно ли что-то перенастраивать?
Когда вы заходите в режим WSL, VS Code может попросить вас переустановить расширения (C++, Makefile Tools и др.) именно «внутри» WSL.
Не пугайтесь, это нормально. Просто зайдите во вкладку расширений, и вы увидите кнопку "Install in WSL: Ubuntu" для тех плагинов, которые мы ставили ранее.
Почему это лучше для ваших тестов и Docker?
Теперь, когда вы будете писать скрипты .sh для тестов, они будут запускаться в «родной» среде. А расширение Bash Debug, о котором мы говорили, наконец-то увидит настоящий путь к интерпретатору Bash и заработает.
