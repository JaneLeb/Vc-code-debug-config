Indent Rainbow ( Для работы с вложенностью )

Что оно делает:
Оно раскрашивает «пустое пространство» (отступы) перед кодом в разные цвета.

Первый уровень вложенности — один цвет.

Второй (внутри цикла) — другой.

Третий — третий и так далее.

Польза:
Когда у вас много циклов for, условий if и скобок {} - это позволяет визуально видеть, какой блок кода к чему относится. Если вы случайно поставите лишнюю скобку или забудете её

CodeMetrics (Контроль сложности)

Для структурного программирования критически важно, чтобы функции не были слишком длинными и запутанными.

Что делает: Оно в реальном времени считает цикломатическую сложность каждой вашей функции и пишет её над заголовком функции (например, Complexity: 2).

Польза: Если цифра становится большой (например, больше 10) и загорается красным — значит, у вас слишком много вложенных if и for. Это сигнал, что пора разбивать код на более мелкие, «чистые» функции по канонам структурного подхода.

TODO Tree (Планирование структуры)

Дейкстра советовал тщательно проектировать программу.
Что делает: Собирает все ваши комментарии вроде // TODO: или // FIXME: в удобный список в левой панели.

Польза: Вы можете сначала разметить структуру будущей программы комментариями (описать шаги алгоритма), а затем постепенно превращать их в чистый код.
Представьте, что ваш код — это большая стройка. TODO Tree — это ваш личный «прораб», который выписывает все недоделки и планы в отдельный аккуратный блокнот.

В процессе написания кода часто возникают мысли: «здесь надо добавить проверку на ноль», «эту функцию я допишу завтра» или «тут баг, надо исправить». Если просто писать это в комментариях, про них легко забыть.

Как это работает на практике:

Вы пишете в коде специальную заметку, например:

    // TODO: Добавить проверку ввода
    // FIXME: Исправить утечку памяти в этом цикле

TODO Tree мгновенно находит эти строки по всему проекту (даже если у вас 50 файлов).
В левой панели VS Code появляется иконка «дерева». Нажав на неё, вы увидите список всех ваших задач.
Клик по задаче в списке — и редактор сам перекидывает вас в нужный файл на нужную строку.

Зачем это нужно именно вам (в контексте Дейкстры и Школы 21):
Проектирование «сверху вниз»: Вы можете сначала создать пустой файл и расписать его структуру одними комментариями // TODO: Шаг 1..., // TODO: Шаг 2.... У вас перед глазами в левой панели будет готовый план на разработку.

Дисциплина: Вы не забудете почистить память или удалить отладочный вывод перед сдачей проекта (просто ставите // TODO: Удалить перед защитой).

Наглядность: Заметки FIXME обычно подсвечиваются ярким цветом прямо в коде, чтобы они «мозолили глаза» и вы их исправили.

Error Lens — внимательные глаза (подсветка ошибок в строке).
Error Lens — это одно из самых популярных расширений для редактора VS Code, которое кардинально улучшает работу с ошибками и предупреждениями в коде. Его главная цель — сделать диагностику кода моментальной и наглядной, устраняя необходимость искать ошибки в нижней панели или наводить мышь на подчеркнутые участки. 

Подробнее: для чего нужен

Мгновенная видимость ошибок (Inline Diagnostics)

Как обычно: VS Code подчеркивает ошибку волнистой линией. Чтобы узнать, что не так, нужно навести курсор.

С Error Lens: Текст ошибки (например, Expected ',' или Variable 'x' is not defined) отображается сразу в конце строки, где она произошла.
Подсветка всей строки

Расширение не просто подчеркивает ошибку, а заливает цветом всю строку, что делает проблему заметной, даже если вы бегло просматриваете код.

Цветовая кодировка: Красный (ошибки), желтый/оранжевый (предупреждения), синий (информация).

Ускорение отладки и разработки

Вы видите проблему в момент написания кода. Не нужно запускать компиляцию или смотреть в панель Problems, чтобы понять, что что-то пошло не так.

Устранение "слепых зон"

В длинных файлах ошибки на других строках легко пропустить. Error Lens выводит их прямо перед глазами. 

Основные преимущества
Экономия времени: Меньше движений мышкой, быстрее исправление.

Понятный контекст: Сразу понятно, какая именно ошибка, без всплывающих подсказок.

Глубокая настройка: Можно настроить цвета, стиль отображения, включить/выключить отображение предупреждений, игнорировать определенные файлы.

Работает "из коробки": Прекрасно интегрируется с линтерами (ESLint, Pylance, TypeScript), уже встроенными в среду. 

Настройка под себя

Если подсветка кажется слишком яркой, в настройках (settings.json) можно изменить её стиль. Например, сделать так, чтобы текст ошибки появлялся только при наведении или сделать фон менее ярким. 

Резюме: Это расширение превращает скрытые диагностические сообщения в явные, что значительно повышает продуктивность, особенно в крупных проектах.

Что такое Dev Containers?

Представьте, что вы хотите программировать в среде Linux (как в школе), но сидите на Windows. Это расширение позволяет VS Code «нырнуть» внутрь изолированного контейнера (Docker), где установлены все нужные инструменты.

Если вам нужно собрать и запустить mini-verter (или похожий проект) в изолированной среде, тогда Dev Containers и Docker — это именно то, что доктор прописал! Это очень крутой навык для "Школы 21", так как там часто проекты проверяются в специфических Linux-окружениях.
Однако, чтобы эта «ракета» взлетела, нужно убедиться, что фундамент готов.

Что нужно, чтобы Dev Containers заработал:

-Docker Desktop: Он должен быть установлен в Windows и запущен (в трее должен быть китенок). Без него расширение в VS Code — это просто пустая кнопка - расширение Dev Containers будет выдавать ошибку при попытке запуска.

-WSL 2: Docker на Windows лучше всего работает через подсистему Linux (Windows Subsystem for Linux).

-Папка .devcontainer: В вашем проекте должна появиться папка с этим названием, а в ней файл devcontainer.json. В нем описывается, какой образ Linux использовать и какие расширения (типа C++ и Makefile) установить внутри контейнера.

Как это меняет работу с Makefile:

Когда вы работаете через Dev Containers: Ваш компилятор cl.exe (Windows) больше не нужен. Внутри контейнера будет работать стандартный линуксовый gcc или clang.

Makefile станет "настоящим" (линуксовым). Вам не нужно будет писать del для очистки, вы будете писать rm -f.
Расширение Makefile Tools будет работать прямо внутри Docker, подхватывая все зависимости.

Ваш следующий шаг для веб-сервера:
Если Docker Desktop уже установлен:

Нажмите на синий значок >< в левом нижнем углу VS Code.

Выберите "Add Dev Container Configuration Files...".

Выберите подходящий образ (например, C++).

VS Code предложит "Reopen in Container". Нажмите это, и вы окажетесь внутри Linux, не выходя из Windows.

Минусы:
Сложность: Чтобы оно работало, на ваш компьютер нужно установить огромную и тяжелую программу — Docker Desktop. Без нее это расширение просто бесполезно.

Путаница: Вам придется разбираться не только в языке Си, но и в том, как настраивать контейнеры, пробрасывать в них папки и настраивать сеть.

Ресурсы: Docker потребляет много оперативной памяти.
Вердикт: Если вы не планируете прямо завтра разворачивать серверы или работать в изолированных Linux-средах на Windows, это расширение можно смело удалять или отключать.
